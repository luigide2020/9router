# M365 Copilot本地Tool执行支持 - 实施计划

## 目标

将M365 Copilot的tool call能力从**Microsoft远程沙箱**迁移到**本地执行**：
- M365 Copilot返回tool调用意图（不执行）
- 9Router将tool_calls返回给下游CLI工具（如codex）
- codex在本地执行tool并返回结果
- 9Router将结果回传给M365继续对话

## 当前问题诊断

### 🔴 关键问题1：会话状态管理缺失

**位置**：[m365-copilot.js:478-481](open-sse/executors/m365-copilot.js)

```javascript
// ❌ 当前实现：每次都生成新的会话ID
const rawHex = crypto.randomUUID().replace(/-/g, "");
const sessionIdHex = rawHex;
const sessionIdUuid = `${rawHex.slice(0,8)}-${rawHex.slice(8,12)}-...`;
const conversationId = crypto.randomUUID(); // 每次都是新的！
```

**问题**：
- 每次请求都创建新的conversationId和sessionId
- M365无法关联之前的tool_call和当前的tool_result
- 多轮对话状态完全丢失

**对比其他provider**：
- ✅ Kiro: 使用 `resolveSessionId()` ([openai-to-kiro.js:558](open-sse/translator/request/openai-to-kiro.js))
- ✅ Antigravity: 使用 `resolveSessionId()` ([antigravity.js:118](open-sse/executors/antigravity.js))
- ✅ Codex: 使用 `resolveSessionId()` ([codex.js:13](open-sse/executors/codex.js))
- ❌ M365: **未使用会话管理器**

### 🟡 问题2：Tool定义转换可能不够精确

**位置**：[openai-to-m365-copilot.js:36-93](open-sse/translator/request/openai-to-m365-copilot.js)

当前将OpenAI tools转换为文本提示词，可能的问题：
- 提示词过于严格（"You CANNOT run commands..."）
- M365可能需要更结构化的方式理解tools
- JSON schema可能未被准确传递

### 🟡 问题3：Tool result回传格式

**位置**：[openai-to-m365-copilot.js:95-98](open-sse/translator/request/openai-to-m365-copilot.js)

```javascript
function buildToolResultPrompt(toolCallId, toolName, result) {
  const resultStr = typeof result === "string" ? result : JSON.stringify(result);
  return `[Tool Result for ${toolName} (id: ${toolCallId})]:\n${resultStr}`;
}
```

可能需要优化：
- 格式可能不够清晰
- M365可能无法正确关联tool_result到之前的tool_call

### 🟢 问题4：执行flags策略

**位置**：[m365-copilot.js:106-119](open-sse/executors/m365-copilot.js)

当前禁用所有执行能力，这是正确的（我们想要本地执行），但可能过于激进。

## 实施步骤

### Step 1: 实现会话状态管理（关键修复）

**文件**：[open-sse/executors/m365-copilot.js](open-sse/executors/m365-copilot.js)

**修改内容**：

1. **导入会话管理器**：
```javascript
import { resolveSessionId } from "../utils/sessionManager.js";
```

2. **修改execute方法**，使用稳定的会话ID：
```javascript
async execute({ model, body, stream, credentials, signal, log }) {
  // ... 现有代码 ...
  
  // ✅ 使用会话管理器生成稳定的conversationId
  const connectionId = credentials?.connectionId || credentials?.email || `${oid}@${tid}`;
  const conversationId = resolveSessionId({
    headers: credentials?.rawHeaders,
    body,
    connectionId,
    scope: "m365-copilot"
  });
  
  // sessionId也使用稳定值
  const sessionIdBase = resolveSessionId({
    headers: credentials?.rawHeaders,
    body,
    connectionId: `${connectionId}:session`,
    scope: "m365-copilot-session"
  });
  
  // 从sessionIdBase生成UUID格式（M365需要UUID格式）
  const sessionIdHash = crypto.createHash("sha256").update(sessionIdBase).digest("hex");
  const sessionIdUuid = `${sessionIdHash.slice(0,8)}-${sessionIdHash.slice(8,12)}-${sessionIdHash.slice(12,16)}-${sessionIdHash.slice(16,20)}-${sessionIdHash.slice(20,32)}`;
  const sessionIdHex = sessionIdHash.slice(0, 32);
  
  // ... 后续使用这些稳定ID ...
}
```

**预期效果**：
- 相同连接的多轮对话使用相同conversationId
- tool_result能正确关联到之前的tool_call
- M365能维护对话上下文

**可选优化**：保留某些安全的执行能力（如搜索），但禁用代码执行：
```javascript
function buildCopilotOptionsSets(enableReasoning = false, disableExecution = false) {
  const sets = [...M365_DEFAULT_OPTIONS_SETS];
  
  if (disableExecution) {
    // 只禁用代码执行，保留搜索等能力
    const codeExecutionFlags = [
      "cwc_code_interpreter", "cwc_code_interpreter_amsfix",
      "cwc_code_interpreter_citation_fix", "code_interpreter_interactive_charts",
      "cwc_code_interpreter_interactive_charts_inline_image",
      "code_interpreter_matplotlib_patching",
    ];
    for (let i = sets.length - 1; i >= 0; i--) {
      if (codeExecutionFlags.includes(sets[i])) sets.splice(i, 1);
    }
  }
  
  return sets;
}
```

**注意**：这个修改需要测试验证，可能不需要。

1. **会话ID是否稳定**：

   - 检查M365返回的文本是否包含 ````json-tool``` 块
   - 如果没有，可能是提示词问题

4. **Tool result回传**：
   - 检查第二轮对话是否使用相同会话ID

## 风险与兼容性

### 风险评估

| 风险 | 影响 | 缓解措施 |
|-----|------|---------|
| 会话ID冲突 | 多用户使用相同ID | 使用connectionId隔离 |
| M365协议变更 | WebSocket格式变化 | 保留原有协议兼容代码 |
| Tool格式不支持 | M365无法理解tools | 提供降级方案（无tool模式） |
| 会话过期 | 长时间对话失效 | 实现会话刷新机制 |

### 向后兼容

- 保留原有的disableExecution逻辑
- 会话管理失败时降级到随机ID
- 不影响其他provider的正常工作

## 总结

**核心问题**：会话状态管理缺失导致多轮tool calling失败

**核心修复**：使用 `resolveSessionId()` 实现稳定的conversationId管理

**预期效果**：
- ✅ M365能正确关联多轮对话
- ✅ Tool calls能正确返回给codex执行
- ✅ Tool results能正确回传给M365
- ✅ 实现本地tool执行而非远程沙箱

**实施优先级**：
1. P0: 会话状态管理（必须）
2. P1: Tool定义和result格式优化（重要）
3. P2: 执行flags调整（可选）
